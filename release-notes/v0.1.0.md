![AI-Infra-Auto-Driven-SKILLS repo](https://files.mdnice.com/user/59/9483093e-9ad2-4c53-ac83-a15f738d3b5b.png)

`AI-Infra-Auto-Driven-SKILLS` v0.1.0 是一次面向 AI Infra / LLM Serving 的 SKILLS 集合发布。它不是 prompt 堆砌，而是把推理框架开发里最容易重复踩坑的流程沉淀成 Agent 可以执行的 playbook：benchmark、capacity planning、torch profiler triage、pipeline analysis、compute simulation、human review、SOTA loop、线上排障，以及模型 PR 历史知识库。

欢迎大家关注和使用：https://github.com/BBuf/AI-Infra-Auto-Driven-SKILLS

如果想使用一个固定版本，可以直接从 v0.1.0 Release 开始：https://github.com/BBuf/AI-Infra-Auto-Driven-SKILLS/releases/tag/v0.1.0

这些 SKILLS 不绑定某一个 Agent runtime。Codex 和 Claude Code 都可以用，目录结构就是普通的 `SKILL.md` skill 目录。以 v0.1.0 为例：

如果只是试用，也可以先只装其中一两个 skill；如果要跑完整 SOTA loop，建议同时安装 benchmark、profiler、pipeline analysis、model PR history，以及 Humanize/RLCR 相关 skill。

# 0x1. Core Skills

这版核心技能主要覆盖下面几类场景：

| Skill | 解决的问题 |
| --- | --- |
| `llm-serving-auto-benchmark` | 对 SGLang、vLLM、TensorRT-LLM 或其它 OpenAI-compatible server 做公平、有限预算的 serving benchmark 搜索。 |
| `llm-serving-capacity-planner` | 从 SGLang/vLLM 启动日志解释 GPU memory、KV cache、request capacity、OOM pressure。 |
| `llm-torch-profiler-analysis` | 统一分析 torch profiler trace，输出 kernel、overlap、fuse opportunity 三张表，并区分 prefill/decode。 |
| `llm-pipeline-analysis` | 继续下钻 forward、layer、kernel timeline，给出代表层、anchor kernel 和 Perfetto 时间范围。 |
| `model-compute-simulation` | 根据模型结构估算 operator shapes、FLOPs、MFU，并把 kernel 和 op 对起来。 |
| `sglang-humanize-review` | 用 2024-2025 SGLang human review 语料来做更接近 maintainer 风格的代码审查。 |
| `sglang-sota-humanize-loop` | 给一个模型和硬件预算，让 SGLang 在固定 workload/SLA 下追平或超过可复现的最强 competitor。 |
| `vllm-sota-humanize-loop` | 同样的 SOTA 闭环，但目标框架换成 vLLM。 |
| `sglang-prod-incident-triage` | 面向线上 serving 问题，从 queue growth、timeout、wrong output、crash、hang 先抽 replay，再决定下一步 debug。 |
| `model-architecture-diagram` | 快速返回 DeepSeek、GLM、Qwen、Kimi、MiniMax、Step、Hunyuan、Qwen3-VL 等模型的公开原始架构图。 |

另外，`model-pr-optimization-history` 现在是一个独立知识库，里面整理了 SGLang/vLLM 的模型优化 PR 历史。SOTA loop 在动手 patch 之前会先查它，避免 Agent 重复试别人已经验证过的方向。

# 0x2. 两个 SOTA Loop

`sglang-sota-humanize-loop` 的核心是先固定公平 benchmark，再把 gap decision、profile、pipeline analysis、patch、revalidate 全部放进同一个 Humanize RLCR 闭环。也就是说，benchmark 表不是装饰品，而是后续每一轮 patch 的起点。

![SGLang SOTA Humanize Loop](https://files.mdnice.com/user/59/2f5360d4-287c-4a24-92fa-525fde210f61.png)

`vllm-sota-humanize-loop` 是同一个思路的 vLLM 版本。它会先拿 vLLM、SGLang、TensorRT-LLM 做同预算搜索，再决定 vLLM 是否真的落后；如果落后，就用 profiler、pipeline analysis、必要时的 NCU 证据去推动 vLLM 源码改动，而不是只靠调 launch 参数。

![vLLM SOTA Humanize Loop](https://files.mdnice.com/user/59/aca581d2-2aa8-4ca9-ab04-30f29b5ab3f8.png)

这两个 loop 有几个共同纪律：

- 不拿 tuned target framework 去比 competitor default，每个框架都要 bounded search。
- 固定模型、精度、GPU 数、workload、SLA 后再谈性能差距。
- profile 先给证据，`llm-pipeline-analysis` 再定位到层和 kernel，最后才 patch。
- kernel 级改动需要 Nsight Compute 证据时，再接 `ncu-report-skill`。
- 每轮都把 benchmark、profile、失败尝试、patch 和复测结果落到 artifact。

# 0x3. OpenAI Goals 和 4 组 Prompt

OpenAI Codex 的 `/goal` 可以理解成线程级的持久目标：它不是让你把 prompt 写得更长，而是让一个 Codex 线程持续围绕同一个完成条件推进。一个好的 Goal 要写清楚结果、验证证据、约束、边界、迭代策略，以及什么情况应该停止并报告 blocker。官方入口见这里：https://developers.openai.com/codex/cli/slash-commands#set-an-experimental-goal-with-goal

这次仓库里放了 4 组 SGLang SOTA prompt，都是为真实 B200/H200 优化任务准备的：

| Prompt | 用法 |
| --- | --- |
| `sglang-sota-b200-prompts.md` | 普通 skill prompt 版本，面向 B200，覆盖 1/2/4/8 GPU 的模型级 SGLang SOTA 任务。 |
| `sglang-sota-h200-prompts.md` | 普通 skill prompt 版本，面向 H200，适合 `ion8-h200` / `ion9-h200` 这类远端验证环境。 |
| `sglang-sota-b200-codex-goal-prompts.md` | B200 的 Codex `/goal` 版本，把结果、证据、约束、清理规则和停止条件写进持久目标。 |
| `sglang-sota-h200-codex-goal-prompts.md` | H200 的 Codex `/goal` 版本，适合长时间 benchmark/profile/patch/revalidate 的持续任务。 |

这些 prompt 里有一些我觉得比较关键的工程约束：开始前要查相关 open PR；workspace 必须干净；benchmark/profile 前要记录 GPU 状态；资源不够就等待或停止；只清理当前模型 cache，不动共享 cache；如果需要提 PR，只能推到允许的 fork；每个优化 PR 都要把 benchmark 和 GSM8K/MMLU 精度表写清楚。

# 0x4. 为什么发这个版本

推理框架优化里，真正贵的不是让 Agent 写几行代码，而是让它知道什么时候该 benchmark、什么时候该 profile、什么时候该怀疑数据不可信、什么时候应该停止。v0.1.0 想解决的就是这个问题：把人的工程纪律压进 SKILL 和 prompt 里，让 Agent 不只是“能跑”，而是按可复现的证据链往前跑。

如果你在做 SGLang、vLLM、TensorRT-LLM、模型适配、serving 排障或性能追踪，这个仓库应该会有帮助。欢迎 Star、试用，也欢迎继续贡献新的 AI Infra SKILLS。
